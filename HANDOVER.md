# AIアシスタントへの引き継ぎ事項 (Context / Handover)

## 1. プロジェクトの背景
このプロジェクトは、ユーザー（imahiyoshiさん）が元々作成し、所有している既存のリポジトリ [my-coffee-studio (GitHub)](https://github.com/imahiyoshi/my-coffee-studio) をベースに開発・改修を行っているものです。
AIがゼロから作成した架空のアプリではなく、ユーザーの実際のデータとプロジェクトです。

## 2. Firebase設定に関する厳重注意
前回、AIアシスタントが「このFirebase設定は他人のもの」と誤認し、新規にFirebaseデータベースを再作成（または再作成しようと）したことにより、本来のユーザーデータ（過去のコーヒー豆の記録など）と接続できなくなる問題が発生しました。

- **`firebase-applet-config.json` などの既存設定はユーザー本人のものです。**
- **絶対に `set_up_firebase` ツールなどを使用して新しいデータベースを勝手に作成・上書きしないでください。**
- Firebaseの接続情報や既存の DB スキーマ、セキュリティルールは元のものを尊重し、エラーが出た場合は新しく作るのではなく、「元のDBを使用する前提でコード側の問題（APIやルールの不整合など）を修正」してください。

## 3. GitHubとの連携について
このセッションは AI Studio のプレビュー環境で操作していますが、最終的にはユーザー自身の GitHub 上のリポジトリへ移行・反映されることが前提です。
- 引き継ぎ時には `checkpoint.json` ファイルなどを利用してコンテキストの引き継ぎを行っていますが、GitHub自体の紐付けは AI Studio の機能（Export to GitHub など）を用いてユーザー自身が行う必要があります。
- コードを変更した際は、エクスポートして GitHub に戻しやすいように、一般的なフレームワークの標準に沿った構成を維持してください。

これらのコンテキストを踏まえて、引き続きユーザーのサポートを行ってください。
